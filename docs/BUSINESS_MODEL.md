# 비즈니스 모델 (Business Model)

---

## 이 서비스의 목적

```
정치를 더 쉽게 만들고, 사회에 가치를 제공하는 서비스.
돈을 버는 게 목적은 아니지만, 돈이 안 되면 가치도 없는 거다.
```

수익 자체가 목표는 아니다.
하지만 누군가 돈을 낸다는 건 그만큼의 가치가 있다는 증거다.
수익은 이 서비스가 진짜 필요한지를 검증하는 지표로 쓴다.

---

## 누가, 왜, 돈은 누가 내냐

```
누가 쓰냐:   강남/마포 30~50대 주민
왜 쓰냐:    우리 동네 행정 공고를 쉽게 알려주고 의견 낼 수 있으니까 (무료)
누가 돈 내냐: 부동산 관련 로컬 업체
왜 돈 내냐:  "강남구 30~50대 + 부동산 관심" 타겟에 도달할 채널이 없으니까
```

---

## BM 구조

```
유저한테는 무료 → 로컬 업체가 광고비를 냄
```

이 서비스의 돈은 유저가 아니라 **광고주**에서 나온다.
유저에게 돈을 받는 순간 "알 권리"라는 공익적 가치가 깨진다.
유저가 많이 모일수록 광고 가치가 올라가는 구조.

---

## 핵심 지표

```
이 서비스는 "매일 여는 앱"이 아니라 "알림 올 때 반드시 여는 앱"이다.
핵심 지표가 DAU가 아니라 "구독자 수 × 알림 오픈율"이어야 한다.
```

| 지표 | 왜 중요한가 | 목표 |
|---|---|---|
| 구독자 수 | 광고 도달 모수 | 1년 내 1,000명, 2년 내 5,000명 |
| 알림 오픈율 | 실제 도달률. 이게 낮으면 구독자 많아도 의미 없음 | 40% 이상 유지 |
| 실제 도달 (구독자 × 오픈율) | 광고주가 보는 숫자 | 매회 2,000명+ (광고 영업 가능선) |
| 이슈 추적 수 | 구독 유지 장치. 추적 중인 이슈가 있으면 해지 안 함 | 구독자 1인당 평균 2개+ |

### 왜 DAU가 아니라 오픈율이냐

```
당근: MAU 1,800만, DAU 높음 → DAU 게임
     매일 새 매물 → 매일 들어올 이유 있음

동네정보: 공고 주 2~3건 → 매일 들어올 이유가 구조적으로 없음
         대신 "내 동네 공고"니까 알림 오면 반드시 열어봄
         → DAU 게임이 아니라 오픈율 게임
```

```
The Skimm: 구독 700만, 오픈율 35% → 매회 245만 도달 → 광고 잘 팔림
동네정보:  구독 5,000, 오픈율 50% → 매회 2,500 도달
          → 로컬 업체 광고로서 충분한 도달
```

오픈율 50%가 가능한 이유:
- 일반 뉴스레터: "세계 경제 어쩌구" → 안 열어도 됨 → 오픈율 20%
- 동네정보: "네가 사는 동네 재건축 공고" → 안 열 수가 없음 → 오픈율 40~50%

### 이슈 추적의 역할

```
DAU를 만드는 기능 ❌ (공고가 수개월에 1번이라 매일 열진 않음)
구독 유지 + 이탈 방지 ✅ ("추적 중이니까 구독 끊으면 안 돼")
장기 리텐션 ✅ (재건축 수년 → 수년간 구독 유지)
```

이슈 추적은 DAU를 만드는 게 아니라,
**"알림 올 때 반드시 여는 사람"을 유지하는 장치**다.

## 수익원

### 1순위: 네이티브 광고 (메인)

```
광고주:  공인중개사, 법무사, 인테리어, 이사업체, 세무사
형태:    공고 카드 사이에 자연스럽게 삽입
```

```
예시:
┌─────────────────────────────┐
│ [공고] 은마아파트 정비계획 변경 │
│ AI 설명: ...                 │
│ 관심 847명                   │
├─────────────────────────────┤
│ [광고] 강남 재건축 전문 법무사  │
│ 조합원 권리분석 무료 상담      │
└─────────────────────────────┘
```

왜 이 광고가 비싸냐:
- 일반 배너 광고: 타겟 불명확, 클릭률 0.1%
- 동네정보 광고: **강남구 + 30~50대 + 부동산 관심 + 지금 재건축 공고 읽는 중**
- 이 정도 타겟 정밀도는 네이버/카카오 광고로도 못 만듦

```
단가 (단계별):
구독자 1,000명:  건당 5~10만원 (테스트 단가)
구독자 5,000명:  건당 15~30만원
구독자 10,000명: 건당 30~50만원
구독자 30,000명: 건당 50~100만원
```

원칙: 공익적 톤을 해치지 않는 범위에서만. 공고당 광고 최대 1건.

### 광고 영업 전략

구독자가 모여도 광고주가 자동으로 오지 않는다. 직접 데려와야 한다.

```
Phase 1 (구독자 1,000명):
- 강남구 법무사/공인중개사 5곳에 직접 연락
- "1개월 무료 테스트" 제안
- 효과 리포트 제공 (노출 수, 클릭 수)
- 1곳이라도 유료 전환되면 → 광고 모델 검증 완료

Phase 2 (구독자 5,000명):
- 기존 광고주 만족도 기반으로 소개 영업
- 광고주용 셀프서비스 페이지 제작
- "강남구 재건축 관심 주민 5,000명에게 도달" 한 줄이면 됨

Phase 3 (구독자 10,000명+):
- 광고주가 찾아옴 (여기부터)
```

핵심: **첫 광고 1건이 가장 어렵다.** 무료로라도 태워보고, 효과 데이터를 만들어야 2건째가 된다.

### 2순위: 선거 시즌 리포트 (시즌 수익)

```
무료 리포트 (브랜딩):
- 후보 비교, 의정활동 분석 → 무료 공개
- 이건 수익이 아니라 구독자 폭발 성장의 엔진

유료 커스텀 리포트 (수익):
- 선거 캠프에서 먼저 연락 오면 맞춤 분석 제공
- 먼저 팔러 가지 않음. 찾아오면 대응.
- 단가: 건당 30~50만원
```

### 3순위: 언론사 제휴 (브랜딩 + 수익)

```
초기:  무료 제공 + "출처: 동네정보" 링크백 (SEO + 인지도)
이후:  구독자 늘면 유료 전환 가능 (월 10~20만원)
목표:  3개 언론사 = 월 30~60만원
```

### 의도적으로 안 하는 것

```
❌ 유료 구독: 알 권리를 유료화하지 않는다
❌ 주민 데이터 판매: 신뢰가 전부인 서비스에서 절대 안 함
❌ 정치 광고: 중립성 훼손
```

## 벤치마크: 비슷한 BM의 성공/실패

### 성공 사례

**당근마켓 — 한국 로컬 광고 유일한 성공**
```
구조: 유저 무료 → 로컬 업체 광고
MAU 1,800만, 매출 연 600~700억
교훈: 중고거래로 "매일 들어올 이유"를 만든 게 핵심
      로컬 광고는 트래픽이 있어야 팔린다
```

**Patch.com — 미국 하이퍼로컬 뉴스**
```
구조: 동네 뉴스 무료 → 로컬 광고
월 방문 2,000~3,000만
교훈: 동네 뉴스 + 로컬 광고는 된다. 단, 에디터 인건비로 적자
      → AI 자동화인 동네정보는 이 실패 원인을 원천 제거
```

**The Skimm — 뉴스레터 광고**
```
구독 700만, 매출 연 300~500억 추정, 오픈율 30~40%
교훈: 타겟 좁고 오픈율 높으면 광고 단가 올라감
      "쉽게 설명한다"는 가치가 동네정보와 같음
```

### 고전 사례

**Nextdoor — 동네 소셜**
```
유저 8,900만인데 연 1,000~1,500억 적자
교훈: 유저만 많으면 소용없고, 자주 들어와야 광고가 팔린다
      DAU가 낮으면 광고주가 Meta/Google 선택
```

**FixMyStreet — 영국 시빅테크**
```
순수 시빅테크로 광고 수익화 성공한 사례는 전 세계적으로 없음
결국 B2G SaaS로 전환
```

### 동네정보에 대한 시사점

```
✅ "로컬 무료 + 광고" 모델은 성립한다 (Patch, 당근이 증명)
✅ AI 자동화로 비용 0 → Patch의 실패 원인 제거
✅ 타겟 정밀 + 높은 오픈율 → 광고 단가 프리미엄 가능

⚠️ DAU 서비스가 아니다. 매일 들어올 이유가 구조적으로 없음
⚠️ 대신 "알림 올 때 반드시 여는 앱"으로 설계해야 함
⚠️ 핵심 지표는 DAU가 아니라 "구독자 × 알림 오픈율"
⚠️ 이슈 추적 = 구독 유지 장치 (DAU가 아니라 이탈 방지)
⚠️ 참여 → 액션 루프 = "다음 알림도 열어볼 이유"를 만드는 장치
```

## 경쟁 우위 / 해자 (Moat)

### 이 서비스가 성공하면, 대기업이 똑같이 만들면 어쩌냐?

```
위협 시나리오:
- 당근이 "동네 행정 소식" 탭 추가
- 네이버가 "동네 공고 알림" 출시
- 호갱노노/아실이 공고 알림 기능 추가
- VC 받은 스타트업이 같은 거 더 잘 만듦

이들은 이미 유저가 수백만~수천만이다.
동네정보가 5,000명 모으는 동안 이들이 움직이면?
```

### 솔직한 답: 정보 서비스로만 보면 해자가 없다

```
공공 API 수집 → 누구나 할 수 있음
AI 요약 → 누구나 할 수 있음
알림 발송 → 누구나 할 수 있음

기능만으로는 방어할 수 없다.
```

### 그런데 대기업이 안 하는 이유

```
1. 시장이 너무 작다
   - "강남구 재건축 공고 알림"의 TAM(총 시장 규모)은?
   - 당근 입장에서 이 기능을 만들 개발 인력 대비 ROI가 안 나옴
   - 대기업은 이런 틈새에 리소스를 안 쓴다

2. 시빅테크는 정치적 리스크가 있다
   - 주민 의견, 찬반, 구의회 전달 → 정치적 이슈가 될 수 있음
   - 당근/네이버 같은 플랫폼은 정치적 논란을 극도로 회피
   - "중립적인 척 해도 왜 이 공고를 먼저 보여줬냐" 공격 가능
   - 대기업은 이 리스크를 절대 안 짊어짐

3. 참여→액션은 순수 정보 서비스와 다르다
   - 공고 수집은 따라할 수 있지만
   - "주민 의견 모아서 구의회 전달"은 플랫폼 성격이 완전히 달라짐
   - 당근이 "재건축 찬반 투표" 기능을 달겠냐? 절대 안 달음
```

### 동네정보만의 진짜 해자

```
해자 1: 참여 데이터의 축적
  - "이 이슈에 847명이 관심, 56건의 의견"
  - 이 데이터는 시간이 지날수록 쌓임
  - 경쟁자가 내일 똑같은 서비스 만들어도 이 데이터는 0에서 시작
  - 주민 입장에서 "이미 847명 있는 곳" vs "새로 0명인 곳" → 네트워크 효과

해자 2: 운영자의 진정성
  - "주민을 위해 이 서비스를 만든 사람"은 대기업이 절대 가질 수 없는 것
  - 당근 CEO가 "강남구 주민을 위해"라고 하면 믿냐? 안 믿어
  - 동네에 사는 사람이, 동네를 위해 만든 서비스 → 이건 못 따라함
  - 특히 정치 브랜딩과 결합되면 운영자 자체가 브랜드

해자 3: 이슈 추적의 시간 자산
  - "은마아파트 재건축 2024년부터 추적 중, 현재 3/7단계"
  - 이 히스토리를 처음부터 쌓아온 서비스 vs 어제 시작한 서비스
  - 시간이 지날수록 기존 서비스의 가치가 올라감
```

### 해자가 약한 시점

```
⚠️ 초기 (구독자 1,000명 미만):
  - 참여 데이터 적음 → 네트워크 효과 없음
  - 이슈 히스토리 짧음 → 시간 자산 없음
  - 이 시기에는 "운영자의 진정성"만이 유일한 해자
  - 즉, 초기에는 해자가 거의 없다

✅ 중기 이후 (구독자 5,000명+):
  - 참여 데이터 축적 → 경쟁자 진입 장벽
  - 이슈 히스토리 수년 → 대체 불가
  - 브랜드 인지도 → "동네 공고 = 동네정보"
```

### 결론

```
이 서비스의 해자는 "기능"이 아니라 "시간"이다.
빨리 시작해서 데이터를 쌓고, 브랜드를 만드는 것 자체가 경쟁 우위.
늦게 시작할수록 해자가 얇아진다.
```

## 수익 시나리오

### 6월 (선거 시즌)

```
광고:               0원 (구독자 부족)
선거 리포트 (무료):  구독자 확보가 수익
커스텀 리포트:       연락 오면 1~2건 × 30만원 = 30~60만원
─────────────────────────────
현금: 0~60만원
가치: 구독자 300~1,000명 확보 (이게 진짜 자산)
```

### 1년 후 (구독자 1,500명 기준, 현실적)

```
네이티브 광고 월 2~3건 × 10만원 = 20~30만원/월
언론사 제휴 (무료 제공):          0원 (브랜딩)
─────────────────────────────
합계: 월 20~30만원
```

### 2년 후 (구독자 5,000명 기준, 낙관적)

```
네이티브 광고 월 4건 × 20만원 = 80만원/월
언론사 제휴 2개 × 15만원     = 30만원/월
선거 시즌 리포트 (연 1회):     100~200만원
─────────────────────────────
합계: 월 110만원 + 시즌 수익
```

### 서울 25개 구 확장 시 (3년 후, 꿈)

```
구독자 50,000명 기준
네이티브 광고 월 10건 × 50만원 = 500만원/월
언론사 제휴 5개 × 20만원      = 100만원/월
─────────────────────────────
합계: 월 600만원
```

> 이 숫자가 중요한 게 아니라, **각 단계에서 광고가 팔리는지**가 중요하다.
> 구독자 1,000명에서 광고 1건이 안 팔리면, 5만 명이어도 안 팔린다.
> 작게 검증하고 키우는 구조.

## 왜 지속 가능하냐

```
뉴닉: 에디터 10명, 인건비 연 15억, 매출 20억 = 적자
동네정보: AI 자동화, 혼자 운영, 비용 월 5만원 = 광고 1건이면 흑자
```

AI 자동화가 핵심 경쟁력이자 수익성의 이유.
뉴닉은 수익 없으면 못 버티지만, 동네정보는 수익 없어도 유지 가능.

## 이 서비스의 진짜 가치

### 사회적 가치

```
- 행정 정보 접근성 향상 (알 권리)
- 시민 참여 허들 낮춤 (보통 사람 → 참여하는 시민)
- 지방 정치를 가깝게 만듦 (구의회가 뭐 하는지 알게 됨)
```

### 개인적 가치 (정치 인프라)

```
구독자 = 지역 주민과의 접점
주민 의견 = 공약의 근거
서비스 운영 = "주민을 위해 일한 사람" 브랜딩
```

| 정치 자산 | 보통 확보 비용 | 동네정보로 확보 |
|---|---|---|
| 지역 주민 연락처 1만 명 | 수천만원 (광고비) | 서비스 구독자 |
| "지역 전문가" 이미지 | 수년간 활동 | 서비스 운영 자체 |
| 주민 니즈 데이터 | 여론조사 건당 수백만원 | 자동 축적 |
| 언론 노출 | PR 비용 | "시빅테크 창업자" 프레임 |

## 이게 안 되는 경우

```
❌ 구독자가 안 모이면 → 광고도 사회적 가치도 정치 자산도 없음
❌ 특정 정치적 입장을 플랫폼 이름으로 표명하면 → 중립성 깨져서 끝
❌ 운영을 꾸준히 안 하면 → 신뢰 잃음 (공고 업데이트 안 되면 끝)
❌ 광고가 공익적 톤을 해치면 → 유저 이탈
```

## Kill Criteria (포기 기준)

**투자와 집착의 차이는 "여기까지 해보고 안 되면 접는다"가 있느냐 없느냐.**

### 0단계 검증 (카드뉴스)
```
GO:   카드뉴스에 댓글/DM으로 "알림 받고 싶다" 5명 이상
KILL: 2주 동안 3개 채널에 올려도 반응 0 → 콘텐츠 방향 전면 수정
```

### 베타 오픈 후 (5월 기준)
```
GO:   구독자 50명 이상 + 알림 오픈율 30% 이상
KILL: 구독자 30명 미만 → 타겟/채널/가치 전면 재검토
```

### 선거 시즌 이후 (7월 기준)
```
GO:   구독자 200명 이상 + 알림 오픈율 40% 이상
KILL: 구독자 100명 미만 → 서비스 유지 여부 결정
      구독자 200명 이상인데 오픈율 15% 미만 → 콘텐츠가 가치 없다는 뜻. 방향 재검토.
```

### 1년 후
```
GO:   구독자 1,000명 이상 + 오픈율 40% 이상 + 광고 또는 제휴 수익 1건 이상
KILL: 구독자 500명 미만 → 축소 운영 또는 중단
      구독자 1,000명 이상인데 수익 0원 → 가치가 돈으로 안 이어지는 구조
      구독자 1,000명 이상인데 오픈율 20% 미만 → 구독만 하고 안 봄. 허수.
      → "좋은 서비스"와 "필요한 서비스"는 다르다. 방향 재검토.
```
