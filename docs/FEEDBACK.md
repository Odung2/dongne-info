## #2 도시계획 "쉬운 설명"이 쉽지 않음

**제보일**: 2026-03-30
**상태**: 🔴 미해결
**심각도**: 최고 (서비스 핵심 가치 직결)

**문제**: 도시계획 공고의 summary에 원본 고시문이 그대로 들어가 있음. "건설부고시 제345호(1979.09.21.)로..." 같은 행정 언어 그대로 노출. 이건 "쉬운 설명"이 아님.

**원인**: cityplan 크롤러에서 고시문 전문(CN)을 summary에 임시 저장한 상태. AI 요약이 아직 실행 안 됨.

**해결 방안**:
1. 도시계획 19건에 AI 요약 실행 (기존 summary를 AI 요약으로 교체)
2. 도시계획 전용 프롬프트 작성 (고시문 전문이 입력이므로 재개발보다 풍부한 요약 가능)
3. 고시번호("건설부고시 제345호")는 요약에서 제거 — AI에게 "법적 근거 조항은 생략하고 핵심만" 지시
4. "나한테 어떤 의미?" 한 줄 추가 — 영향 범위, 추천 액션 포함

### 추가 논의: 분류별 프롬프트 분리

**결론: 분리하는 게 맞다.** 이유:
- 재개발: 입력이 짧음 (사업명, 위치, 면적, 분류). 요약은 "이게 뭔지" + "어느 단계인지".
- 도시계획: 입력이 풍부 (고시문 전문 수천 자). 요약은 "핵심이 뭔지" + "누구에게 영향인지".
- 추후 장르 확장(교통, 교육 등) 시에도 각 장르별 프롬프트가 필요.

**구조 설계**:
```
service/summary.go
  - SummarizeUnsummarized() → 공고 type에 따라 다른 프롬프트 선택
  - summarizeRebuild()      → 재개발/재건축 전용 프롬프트
  - summarizeCityplan()     → 도시계획 전용 프롬프트
  - summarizeGeneral()      → 기본 프롬프트 (새 장르 추가 시 임시 사용)
```

### 추가 논의: 고시번호 없이 AI가 제대로 이해하나?

**가능하다.** 고시번호는 법적 참조용이지 내용 자체가 아님. 고시문에는 고시번호 외에 핵심 정보(뭘 하겠다, 어디서, 왜)가 다 있어. AI에게 "법령 인용과 고시번호는 무시하고 실제 내용만 요약해"라고 지시하면 됨.

## #3 정비사업 카테고리 문제

**제보일**: 2026-03-30
**상태**: 🔴 미해결
**심각도**: 중간

**문제**: "정비사업"은 우리 크롤러가 만든 분류명. 원래는:
- 도시환경정비사업구역 (28건) — 상업지역 재개발과 유사
- 도시환경정비사업지구 (5건)
- 주거환경개선사업구역 (1건)

유저에게 "정비사업"이라는 말은 의미 없음. 필터에도 안 나옴.

**해결 방안**:
- "도시환경정비사업" → "도심 재개발"로 표시 (실질적으로 상업지역 재개발)
- 또는 필터를 "재개발/재건축/도시계획" 3개로 유지하되, 도시환경정비사업을 "재개발"에 포함
- 주거환경개선사업은 "주거환경개선"으로 별도 표시하거나 "재개발"에 포함

## #4 원문 링크가 작동하지 않음

**상태**: 🔴 미해결
**심각도**: 중간

**해결 방안**:
- 재개발: "원문 보기 (준비 중)" 표시 + "서울시 정비사업 정보몽땅에서 검색하세요" 안내 문구
- 도시계획: 고시문 전문(CN)이 DB에 있으므로 상세 페이지에서 접기/펼치기로 직접 보여주기
- 링크가 작동하지 않을 때 fallback 안내 표시

## #5 태그(뱃지)의 의미가 불명확

**제보일**: 2026-03-30
**상태**: 🔴 미해결
**심각도**: 중간

**문제**: "결정+지적", "변경", "실시" 등 행정 용어를 그대로 뱃지로 표시. 일반인에게 의미 불명.

**해결 방안**: 뱃지에 호버(마우스)/클릭(모바일) 시 설명 툴팁 추가.

```
뱃지 → 툴팁 설명:
"신설"      → "새로운 구역이나 사업을 처음으로 지정한 거예요"
"변경"      → "기존에 정해진 계획이나 구역을 수정한 거예요"
"폐지"      → "기존 사업이나 구역 지정을 취소/해제한 거예요"
"결정+지적" → "도시계획을 확정하고 공식 지도에 표시한 거예요"
"결정"      → "도시계획을 공식적으로 확정한 거예요"
"실시"      → "확정된 계획을 실제로 착공할 수 있게 승인한 거예요"
```

## #6 "나한테 어떤 의미?" 추가

**제보일**: 2026-03-30
**상태**: 🔴 미해결
**심각도**: 높음

**문제**: 공고를 읽어도 "그래서 나한테 뭐?" 반응. 어떤 영향이 있는지, 내가 뭘 해야 하는지 모름.

**해결 방안**: AI 요약에 "의미/영향" + "추천 액션" 필드 추가.

```json
{
  "summary": "쉬운 설명 2~3문장",
  "stage": "진행 단계",
  "impact": "이 근처 거주자라면: 향후 재건축 조합 설립 시 참여 기회가 생겨요",
  "action": "관심 있다면 구청 도시계획과에 상세 내용을 문의해보세요 (☎02-xxx-xxxx)"
}
```

주의: impact도 예측이 아닌 팩트 기반이어야 함. "집값이 오른다" 금지. "재건축 진행 시 이주/보상 절차가 시작될 수 있어요" 정도.

## 추가 논의: 진행 단계 기준

### 재개발/재건축 7단계 (법적 근거: 도시 및 주거환경정비법)

```
①기본계획 수립 → ②정비구역 지정 → ③조합 설립 → ④사업시행인가
→ ⑤관리처분인가 → ⑥착공 → ⑦입주
```

이 7단계는 법률에 정의된 공식 절차. AI가 공고의 사업명/분류에서 판별:
- "정비구역 지정" 공고 → 2단계
- "조합설립인가" 공고 → 3단계
- "사업시행계획 인가" 공고 → 4단계

### 도시계획 진행 단계 (재개발과 다름)

```
①입안 (계획 초안) → ②주민 열람/공고 → ③위원회 심의 → ④결정 고시 → ⑤지형도면 고시
```

도시계획 공고의 ANCMNT_TYPE에서 판별:
- "결정" → 4단계
- "결정+지적" → 4~5단계 (결정과 지형도면 동시)
- "실시" → 실시계획 (착공 준비)

현재 StageBar는 "N/7단계" 패턴만 파싱하는데, 도시계획은 "N/5단계"로 별도 처리 필요.
