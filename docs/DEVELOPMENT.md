# 개발 플로우 & 우선순위 (Development Flow)

> 하루 3시간 × 65일 = 195시간
> 목표: 6월 3일 지방선거 전 베타 오픈

---

## 핵심 원칙

```
코드보다 검증이 먼저.
카드뉴스 1장의 반응이 코드 1,000줄보다 중요하다.
```

---

## 타임라인

```
0주차    │ 콘텐츠 검증 (서비스 없이 카드뉴스로 반응 테스트)
1~2주차  │ 데이터 파이프라인 구축
3~4주차  │ 웹 MVP 개발 (참여 기능 포함)
5주차    │ 베타 오픈 + 단지별 침투
6월 전   │ 선거 특집 + 무료 리포트
6월 3일  │ 지방선거 → 트래픽 피크
```

---

## 0단계: 콘텐츠 검증 (이번 주)

**서비스 개발 전에 수요 확인**

```
Day 1-2: 씨앗 콘텐츠 제작
- 타겟 단지 5개 선정 (은마, 대치쌍용, 개포주공, 마포래미안, 성산시영)
- 각 단지별 "재건축 어디까지 왔나" 카드뉴스 1장 제작
- 하단에 "새 공고 나오면 알려드릴까요?" → 구독 폼 링크

Day 3-5: 채널 배포 + 반응 측정
- 네이버 부동산 카페 (강남/마포) 글 올리기
- 단지별 오픈카톡방에 공유
- 당근마켓 강남/마포 지역 게시
- 지인 10명에게 보내고 각자 10명에게 공유 요청
```

### GO / KILL 기준
```
GO:   댓글/DM으로 "알림 받고 싶다" 5명 이상 → MVP 개발 시작
KILL: 2주 동안 3개 채널에 올려도 반응 0 → 콘텐츠 방향 전면 수정 후 재시도
```

---

## 1단계: 데이터 파이프라인 (1~2주차)

### 구현 순서

**Day 1-2: Supabase 세팅 + DB 스키마**
```
- Supabase 프로젝트 생성
- schema.sql 실행 (announcements, subscribers, reactions, comments)
- Go에서 DB 연결 테스트
```

**Day 3-4: 재개발 크롤러 완성**
```
- 현재 main.go → crawler/rebuild.go로 분리
- 환경변수에서 API 키 읽기 (SEOUL_API_KEY)
- 강남구/마포구 데이터 DB 저장
- 신규 데이터 감지 로직 (source_id 중복 체크)
```

**Day 5: 도시계획 결정고시 크롤러 추가**
```
- upisCityplan API 연동 (용도지역 변경, 지구단위계획 등)
- 재개발 크롤러와 같은 파이프라인 구조로 통합
```

**Day 6: 자치구 공고 크롤러 추가**
```
- 강남구/마포구 구청 홈페이지 고시·공고 크롤링
- RSS 있으면 RSS, 없으면 HTML 파싱
- ★ 이걸 해야 "공고 없는 날"이 없어짐
```

**Day 7-8: Claude API 요약 연동**
```
- 행정 언어 → 쉬운 설명 변환
- 진행 단계 표시 (전체 절차 중 어디인지)
- 관련 과거 사례 연결 (같은 유형의 다른 지역 공고)
- 프롬프트 튜닝
- ⚠️ 집값 예측 절대 안 함 → 팩트 요약만
```

**Day 9: 스케줄러**
```
- 매일 오전 9시 자동 실행
- 에러 핸들링, 로깅
```

### 테스트 기준
```
✅ 3개 데이터 소스에서 강남구/마포구 데이터 수집됨
✅ 하루 평균 5건 이상 공고 확보 (콘텐츠 빈도 검증)
✅ 신규 공고 감지 작동
✅ Claude API 팩트 요약 생성됨 (예측 문구 없음 확인)
✅ 스케줄러 24시간 연속 실행
```

## 2단계: 웹 MVP (3~4주차)

### 구현 순서

**Day 10-12: Next.js 기본 구조**
```
- 프로젝트 세팅
- 구별 공고 목록 페이지 (/gangnam, /mapo)
- 공고 상세 페이지
```

**Day 13-14: UI 핵심 기능**
```
- 공고 카드 컴포넌트
  - 유형 뱃지 (신설/변경/폐지)
  - 진행 단계 표시 (예: "7단계 중 2단계")
  - AI 쉬운 설명 본문
  - 관련 사례 링크
  - 원문 링크
- 구 선택 필터
```

**Day 15-16: 참여 기능 (핵심 차별점)**
```
- [📌 관심 있어요] 버튼 (비로그인 가능, session 기반)
- 관심 수 실시간 표시
- 카카오톡 공유 버튼 (공고 카드 형태)
- "우리 아파트 이웃 N명이 보고 있어요" 표시
```

**Day 17-18: 알림 구독**
```
- 이메일 구독 폼
- 카카오 알림톡 연동 (선택)
- 구독자 DB 저장
```

### 테스트 기준
```
✅ 강남구/마포구 공고 목록 보임
✅ AI 요약 읽기 쉬움 (지인 5명 테스트)
✅ 관심 표시 버튼 작동
✅ 카카오톡 공유 작동
✅ 이메일 구독 작동
✅ 모바일 화면 정상
```

## 3단계: 베타 오픈 + 단지별 침투 (5주차)

### 배포
```
- Go 백엔드: Railway 배포
- Next.js: Vercel 배포
- 도메인: dongne.info 또는 유사
```

### 유저 확보: 단지별 타겟팅

```
핵심: "강남구 전체"가 아니라 "은마아파트 재건축" 하나를 판다

1. 타겟 단지 5개 선정 (은마, 대치쌍용, 개포주공, 마포래미안, 성산시영)
2. 단지별 "재건축 진행 현황" 카드뉴스 업데이트 (0단계에서 만든 것)
3. 네이버 아파트 카페에 공유 (강남/마포)
4. 단지별 오픈카톡방에 직접 공유
5. 당근마켓 강남/마포 지역 게시
6. 지인 10명 베타 테스트 → 각자 10명에게 공유 요청
```

### 바이럴 설계
```
- "우리 아파트 관련 공고 나왔대" → 아파트 단톡방에 공유
- "이거 관심 표시 해봐" → 관심 수 올라가는 게 공유 동기
- "우리 아파트 이웃 N명이 보고 있어요" → 이웃 효과
```

### 테스트 기준
```
✅ 실제 유저 100명 방문
✅ 구독자 50명 확보
✅ 특정 단지 1곳에서 관심 표시 10회 이상 (밀도 검증)
✅ 주 3회 이상 알림 발송 가능한 콘텐츠 볼륨 확인
✅ 크롤러 무중단 1주일 운영
```

## 4단계: 선거 특집 (6월 전)

### 추가 기능
```
- 지방선거 후보자 페이지
- 후보자 의정활동 성적표 (현직 구의원인 경우)
- 우리 구 후보 재개발 입장 비교
- 투표소 찾기 연동
```

### 무료 리포트 배포
```
- "강남구 구의원 후보 vs 현직, 재개발 입장 비교" 리포트
- 무료 공개 → 브랜딩 + 구독자 폭발 성장
- 언론사도 가져다 씀 → 무료 홍보
- 유일하게 정리된 곳 = 동네정보 → 독점 콘텐츠
```

### 홍보
```
- 지역 언론사 보도자료
- SNS 바이럴 (카드뉴스 형태)
- 연락 오는 선거 캠프에 커스텀 리포트 대응
```

## 유저 확보 목표

```
4월: 구독자 50명 (카드뉴스 + 수동 홍보)
5월: 구독자 300명 (MVP + 입소문)
6월: 구독자 1,000명 (선거 리포트 바이럴)
```

> 첫 유저 100명은 "제품"이 데려오는 게 아니라 "발품"이 데려온다.
> 네이버 카페 글 10개, 오픈카톡방 5개, 당근마켓 3번, 지인 20명.
> 이게 첫 100명의 전부. 제품이 아무리 좋아도 이걸 안 하면 0명.
